# 06. 밸런스 노트

## 개요

ALL-IN의 밸런싱 히스토리와 Python 시뮬레이션 검증 결과 기록.
카드 수치, 매치업 의도, 실제 결과를 비교하며 조정한 과정.

## 밸런스 목표

| 항목 | 목표치 |
|---|---|
| 평균 게임 길이 | 3~6턴 (약 3분) |
| 직업별 종합 승률 | 50% ± 7% |
| 의도된 천적 매치업 승률 | 60% ± 10% |
| 첫 턴 즉사 콤보 빈도 | 5% 미만 |
| 카드별 사용률 편차 | 20~80% (극단적 쓰레기/과사용 방지) |
| 풀베팅 빈도 | HP 30 미만에서 급감 |

### v1 (폐기) — 마나 시스템 기반

초기 디자인. HP 20, 마나 3/턴, 베팅은 마나로.

**문제점:**
- 자원 2종(HP/마나)으로 규칙 복잡
- 마나 베팅이 "이번 턴 손해" 수준이라 도파민 약함
- 시뮬 결과 평균 10턴 이상 길어짐 (컨트롤러 거울전 17턴)

**폐기 사유:** 사용자 피드백 — "HP를 직접 베팅하는 게 더 도박스럽다"

## v2 시뮬레이션 히스토리

### 1차 시뮬 (2,000판/매치업)

**카드 수치 초기안으로 진행한 결과.**

| 매치업 | 승률 | 평균 턴 |
|---|---|---|
| 광전사 vs 도박사 | 광전사 23% | 3.3t |
| 광전사 vs 컨트롤러 | 광전사 65% | 6.4t |
| 도박사 vs 컨트롤러 | 도박사 94% | 4.2t |

**직업별 종합 승률:**
- 광전사 44%
- 도박사 85% (극심한 OP)
- 컨트롤러 21% (망가짐)

**발견된 핵심 문제:**

1. **첫 턴 시그니처 풀콤보** — 도박사가 첫 턴에 G14+G15+G6 콤보로 145뎀 일격
2. **G6 카운팅 12.77뎀 평균** — 덱 사이즈 보너스 너무 큼
3. **도박사 크리 카드 전반 OP** — G15 올인 ×5 배수로 250뎀 이론치
4. **컨트롤러 점진적 운영이 한 방에 무너짐**

### 수정 사항

- G15 올인: 베팅 상한 20→12, 크리 배수 ×5→×3
- G14 마크된 운명: "이번 턴 모든 크리" → "다음 1장 크리"
- 시그니처 3턴 이후 사용 가능 룰 추가
- 같은 턴 시그니처 1장 제한 추가
- W14 무효 선언: 2턴 → 3턴 강화
- W6 인내의 반격: 캡 25→35, /4 → /3
- 광전사 주력 카드 데미지 상향 (B1 18, B2 25, B3 22)
- 베르세르크/분노 보너스 데미지 +10 → +8

### 2차 시뮬 (3,000판/매치업, 수정 후)

| 매치업 | 승률 | 평균 턴 |
|---|---|---|
| 광전사 vs 도박사 | 광전사 26% | 3.7t |
| 광전사 vs 컨트롤러 | 광전사 73% | 6.5t |
| 도박사 vs 컨트롤러 | 도박사 96% | 4.7t |

**직업별 종합 승률:**
- 광전사 49% (정상화)
- 도박사 85% (여전히 OP)
- 컨트롤러 16% (더 악화)

**추가 발견:**

- 평균 턴 수 3~6턴 정상 범위 진입
- 광전사 정체성은 살았으나 천적이어야 할 도박사를 못 이김
- 컨트롤러는 부운 없이는 구조적으로 열세

## 현재 미해결 이슈 및 가설

### 이슈 1: 도박사 vs 컨트롤러 매치업

**현상**: 도박사 96% 승률  
**의도**: 컨트롤러 60% 승률  
**가설**: 
- 컨트롤러는 "상대를 말려 죽이는" 전략이라 게임이 빠르면 자동 패배
- 부운 BN03(회복의 가호), BN01(강철 심장) 등으로 장기전 유도 시 반전 가능

**검증 계획**: 부운 시스템 시뮬에 통합 후 재측정. 개선 안 되면 컨트롤러 W6 추가 버프 검토.

### 이슈 2: 광전사 vs 도박사

**현상**: 광전사 26% 승률  
**의도**: 광전사 60% 승률  
**가설**: 
- AI 휴리스틱의 한계 — 광전사 "함성 → 피의 광기 → 처형" 콤보를 AI가 못 살림
- 실제 사람은 콤보 의식적으로 짜므로 승률 상승 예상

**검증 계획**: CLI 프로토타입으로 친구들과 직접 매치 → 인간 vs 인간 매치업 데이터 수집.

### 이슈 3: 컨트롤러 거울전 평균 11.9턴

**현상**: 서든데스 없으면 17턴까지 늘어지는 경우 있음  
**원인**: 양쪽 모두 방어/회복 위주라 결정타 못 때림  
**해결**: 미러전 금지로 구조적 해결됨 (v2에서 확정)

## AI 시뮬레이션의 한계

실제 밸런싱은 AI 시뮬만으로 완성 불가. 다음 항목은 인간 플레이로만 검증 가능:

| 검증 불가 항목 | 사유 |
|---|---|
| 블러프/읽기 심리전 | AI는 단순 확률로만 판단 |
| 콤보 빌드업 의사결정 | 사람은 n턴 후를 보고 설계 |
| 체감 재미/도파민 | 정량화 불가 |
| UI 직관성 | 시뮬 범위 밖 |
| "이번 판은 어떤 빌드?" 메타 고민 | 부운 + 직업 조합 선택 |

**결론**: 시뮬레이션은 극단적 OP/쓰레기 카드 잡기에 유효. 세부 밸런스는 플레이 테스트로 완성.

## 밸런스 조정 원칙

### 해야 할 것

- 극단적 OP/쓰레기 카드는 즉시 수정
- 시그니처 스케일링은 보수적으로 (첫 턴 콤보 방지)
- 부운으로 커버할 수 있는 매치업 격차는 유지 (빌드 다양성의 재미)
- 사람 테스트 결과를 AI 시뮬보다 우선

### 하지 말 것

- 한 번에 여러 카드 동시 수정 (인과 추적 어려움)
- 시뮬 수치 맹신 (특히 AI가 단순할수록 편향)
- 거울전 밸런싱 (미러 금지로 무효)
- 모든 매치업 50:50 강제 (의도된 천적 관계는 유지)

## 2026-04-17 수정 — 도박사 패시브 변경

### 변경 내용
- 이전: `gambler_passive_options: 4` (부운 선택지 3 → 4)
- 현재: `gambler_reroll_count: 1` (제시된 3개 전부 버리고 3개 재추첨, 1회 한정)

### 변경 사유
"선택지가 많다"는 추상적 장점보다 "내가 원할 때 한 번 뒤집는다"의 **적극적 능동감**이 직업 정체성("운명의 총아")에 더 맞음. 리롤은 순수 도박(결과가 더 나빠질 수도 있음) — 컨셉 강화.

### 밸런스 영향 가설
- 이전(4개 중 1): 대략 모든 부운을 +33% 더 많이 볼 수 있는 확률. 특정 카운터 부운(BN02, BN03) 접근성 증가.
- 현재(리롤): 초기 3개가 쓸 만하면 리롤 안 쓰고, 심각하게 안 맞을 때만 사용. 기대 풀에서 보이는 부운 양은 비슷하거나 조금 적지만 "피해야 할 제시" 에서 벗어날 수 있음.
- 시뮬 검증 필요 (태스크 10 시뮬레이션이 아직 미구현이라 프로토 플레이로 관찰 예정).

## 디자인 문서 vs 데이터 불일치 기록

### 2026-04-17 — 카드 카테고리 집계 (CLI Task 2 진행 중 발견)

**이슈**: `docs/03-cards.md` 본문 및 요약표는 각 직업 "공격 카드 8장 / 방어·유틸 5장 / 시그니처 2장" 이라고 명시.
그러나 `data/cards.json` 의 `category` 필드 기준 집계는:

| 직업 | attack | defense | utility | signature |
|---|---|---|---|---|
| 광전사 | 7 | 2 | 4 | 2 |
| 도박사 | 7 | 2 | 4 | 2 |
| 컨트롤러 | 8 | 1 | 4 | 2 |

- 광전사: **B6 광전사의 함성(utility)**, **B7 고통 흡수(defense)** 가 공격이 아닌 것으로 분류
- 도박사: **G5 더블 다운(utility)** 이 공격이 아닌 것으로 분류
- 컨트롤러만 문서 분류와 JSON category 일치

**해석**: 디자인 문서의 "공격 카드 8장"은 **공격 라인에 속한 카드 묶음**(공격 지원 유틸 포함)을 의미하는 걸로 보이며, JSON category 는 엄격한 실제 효과 분류. 엔진 구현은 JSON 을 기준으로 함 (예: "공격 카드만 서치" 같은 B12 도끼 갈기 효과 처리 시 category=attack 기준).

**결정**: 데이터 수정 없음. 본 파일에 기록만 남김. 문서 본문의 카운트 수치는 서술적 표현으로 간주.

**영향**: 없음. 단 B12 도끼 갈기 "공격 카드 서치" 구현 시 category=="attack" 으로 필터링하면 광전사는 7장 풀에서만 검색됨.
